Week 1

Definitions

  1. Protocol:
    • The way to interact with server, define the behaviour server nees to take
  2. HTTP:
    • A client-server protocol: always initiated by client
    • Why use HTTP on mobile side:
      • Already used in many browser based clients
      • Many existing libraries to use if continue with HTTP
      • Data marshalling is common: HTTP -> other format you need
      • Existing load balancer to reuse
  3. Classic Java application
  1. Cloud Java application

HTTP Request methods

  • GET

    • Important Get things client want

  • POST

    • Important Send a lot of data to server to store/do something else

  • PUT

    • Ask server to store some data contained in the request to server

  • DELETE

    • Ask server to delete things on server

HTTP Request Anatomy

==Necessary==:

  • Request line -> request method + resource (path)
  • Headers -> extra infomation to help server process the request
    • Language type
    • Content type

Optional:

  • Body: core data sent to server to process the request

Note: If Header is not provided while Body is, the server is able to process client’s request without giving back in the correct format/way. But if Body is not included and needed by the server, the request won’t be processed.
